Agenda in detail

NaaS project overview

The broad aim of the NaaS project is to investigate in-network line rate processing that can help data centres where their performance is network bound. In particular this involves several elements:

  • A on path application specific middlebox that can process application data at line rate.
  • Offloading appropriate processing to fast dedicated hardware elements such as the NetFPGA boards where appropriate.
  • Routing and filtering streams as appropriate using carefully designed Software Defined Network (SDN) techniques.

Progress to date: NetAgg box

This is an implementation of the first stage of the NaaS vision, an on-path middlebox that can speed up processing for Hadoop and Solr at line rate. This improves the processing power of these technologies and reduces the burden they put on the data centre network.

Mirage/OpenFlow

OpenFlow is the best-known and most widely deployed SDN technology. Work within the project has implemented a lightweight switch capable of OpenFlow version 1.4 (the latest version at the time of writing) using unikernels within the Mirage/Xen environment. This allows the deployment of an extremely lightweight software switch.

NaaS box: application specific middlebox

The project will generalise the NetAgg work to produce a middlebox design that will allow a wide variety of middlebox tasks (both traditional and novel) to be easily aided with in network processing. A specifically designed high-level language will allow developers to simply code the middlebox task appropriately but will restrict the capabilities to ensure that their designs can run at line rate in a high-speed network environment (10G and above). A brief description of the plans for this will be followed by a general discussion about the best ways to proceed.

NetFPGA DMA for high-performance networking and acceleration

The goal of this project is to make use of NetFPGA for both NIC and general-purpose acceleration by means of high-performance/PCI-efficient DMA engine. The entire hardware (DMA engine) and software (device driver) are being developped from scratch and open-sourced for the current and next generation of NetFPGA boards. The new DMA/driver can be used for high-performance packet processing and offloading of compute-intensive application functions. This project expects its open implementation to help extend NetFPGA to accommodate various application-aware middlebox functionality beyond NIC.
